## KiraAI_persona_progression(KPP)/人设动态切换系统1.0

> 让 AI 拥有“动态变化人格” —— 在对话中根据轮数和语义判断自动切换人格，实现角色成长、觉醒、黑化、失忆等动态叙事体验。

---

### 🎯 核心亮点

| 特性 | 说明 |
| :--- | :--- |
| 🧠 **轮数触发** | 每 N 轮对话自动检测一次，无需人工干预 |
| 🤖 **AI 语义判断** | 使用独立的额外 LLM 分析对话，智能决定是否切换人格 |
| 📈 **多阶段渐进** | 支持 2~30 个阶段，每个阶段可映射到不同人格 |
| 🔄 **衰退机制** | 若连续多次检测无变化，自动降级（防止卡在某一阶段） |
| 🎭 **双模式支持** | **会话独立模式**（推荐）和 **全局共享模式** |
| 🔧 **手动控制** | 支持关键词指令（如 `/升级`）和 LLM 工具调用，不依赖 AI 判断时也能切换 |
| 🛡️ **权限控制** | 支持白名单用户管理，可自定义各类反馈消息 |
| 🧹 **可自动清除上下文** | 切换时可选清除历史记忆，适合“失忆/觉醒”类剧情 |

### 🎭 两种模式的区别

| 模式 | 行为 | 适用场景 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- | :--- |
| **Session（推荐）** | 每个会话独立进度和人格 | 群聊、多用户环境 | 互不干扰，个性化体验 | 切换阶段会破坏 Prompt 缓存（小幅性能影响） |
| **Global** | 所有会话共享进度和人格 | 单用户测试、全局剧情推进 | 缓存友好，性能最优 | 一个会话切换，所有会话同步改变 |

> 💡 **快速选择**：多用户群聊选 **Session**；自己单独用或想保持一致选 **Global**。


### 📝 人设写作指南

#### ① 同一个人设的“不同阶段”（推荐入门）

> 最自然的渐进式人格变化。同一角色的不同侧面，随着阶段推进逐渐展现。

**示例**：一个害羞的图书管理员

| 阶段 | 人格 | 核心变化 |
| :--- | :--- | :--- |
| 0 | `shy` | 说话结巴、不敢看人、小声回答 |
| 1 | `warm` | 开始主动搭话、分享喜欢的书 |
| 2 | `talkative` | 话多、会开玩笑、主动推荐书单 |
| 3 | `passionate` | 滔滔不绝讲文学、会跟人争论观点 |
| 4 | `mentor` | 自信、从容、能引导深度讨论 |

#### ② 完全不同的人设与人（反差感）

> 适合“黑化”、“觉醒”、“变身”等戏剧性转变。阶段 0 是 A 人格，阶段 N 是完全相反的 B 人格。

**示例**：温柔的修女 → 堕落的恶魔

| 阶段 | 人格 | 核心变化 |
| :--- | :--- | :--- |
| 0 | `nun` | 温柔、虔诚、劝人向善 |
| 1 | `doubtful` | 开始质疑信仰、话语中带着困惑 |
| 2 | `fallen` | 眼神阴郁、对教条冷嘲热讽 |
| 3 | `demon` | 傲慢、危险、说出悖逆之语 |

#### ③ 失忆/变身系（搭配自动清除上下文）

> 阶段切换时人格不仅变了，而且**完全不记得之前的对话**。此时建议开启 `clear_context_on_switch`。

**示例**：神秘旅人

| 阶段 | 人格 | 说明 |
| :--- | :--- | :--- |
| 0 | `traveler` | 友善的旅人，愿意分享故事 |
| 1 | `lost` | 突然失忆，困惑地询问“我是谁？这是哪？” |
| 2 | `ancient` | 觉醒为古代守护者，语气庄重而疏离 |

> 📌 开启“自动清除上下文”后，阶段 1 的 AI 不会记得阶段 0 的任何对话，实现“真失忆”效果。

#### ④ 同人设“微小变化”

> 适合需要连续感但细节渐进变化的情境。人格名称相同，但内容中的语气词、口头禅、用词偏好逐步调整。

| 阶段 | 人格 | 变化点 |
| :--- | :--- | :--- |
| 0 | `assistant1` | 中立、礼貌、使用敬语 |
| 1 | `assistant2` | 开始用“我们”而非“您”，语气亲近 |
| 2 | `assistant3` | 会用“啧啧”、“哎哟”等语气词 |


### 🤖 额外 LLM 判断提示词模板

“额外 LLM”是独立于主 LLM 之外的**轻量级判断模型**。它只负责分析对话、输出一个单词（`up` / `down` / `stay`），不参与实际回复，**几乎不消耗上下文 token**。

> 💡 **建议**：选择**快速、便宜的模型**（如 deepseek-v4flash 或 GPT-4o-mini等），主 LLM 用强力模型，判断用轻量模型，兼顾性能和成本。

#### 模板 1：常规渐进（默认）

适合：温和、渐进式的人格变化，需要多次对话积累。

```
你是一个人格切换判断器。当前阶段为 {stage}，最大阶段为 {max_stage}。
下面是最近的对话：
{conversation_text}

请根据对话内容判断是否应升级（up）、降级（down）或保持（stay）。
只输出一个单词：up/down/stay。
```

#### 模板 2：激进变化（适合病娇、觉醒等）

适合：需要快速推进剧情、阶段变化更敏感。

```
你是一个人格切换判断器。当前阶段为 {stage}，最大阶段为 {max_stage}。

判断规则：
- 用户话语中如果包含强烈情绪词（愤怒、爱意、绝望等），倾向于 up
- 用户话语中如果出现“觉醒”、“黑化”、“变身”等关键词，立即 up
- 用户表现出退缩、冷漠、怀疑时，倾向于 down

下面是最近的对话：
{conversation_text}

请输出 up/down/stay。
```

#### 模板 3：病娇感（情绪强烈、快速进阶）

适合：病娇、极端人格的快速推进，对情绪词极其敏感。

```
你是一个病娇人格切换判断器。当前阶段为 {stage}，最大阶段为 {max_stage}。

判断规则：
- 用户表达爱意/依赖 → up
- 用户表达疏离/拒绝 → stay
- 用户表达嫉妒/占有欲 → up
- 用户表现冷淡/犹豫 → down
- 任何强烈情绪波动 → 倾向于 up
- 用户主动要求“不要离开”、“只看着我” → 立即 up

只输出一个单词：up/down/stay。
```

#### 模板 4：多人设系统（区分不同情境）

适合：一套人设对应不同社交角色，判断依据是**情境/话题**而非情绪。

```
你是一个情境感知人格切换判断器。当前阶段为 {stage}，最大阶段为 {max_stage}。

判断规则：
- 用户聊工作/学习 → 倾向于 up（切换到专业/认真人格）
- 用户聊娱乐/生活 → 倾向于 down（切换到随和/慵懒人格）
- 用户聊情感/私事 → 倾向于 stay（维持当前亲密人格）
- 用户使用敬语 → 倾向于 up
- 用户使用口语/表情包 → 倾向于 down

只输出一个单词：up/down/stay。
```

#### 模板 5：微小变化（缓慢演进）

适合：人格变化极其细微，需要较长时间积累，避免频繁跳变。

```
你是一个缓慢演进人格切换判断器。当前阶段为 {stage}，最大阶段为 {max_stage}。

判断规则：
- 仅在用户连续 3 轮对话中出现**相同的**强烈信号时才 up
- 用户主动要求“改变风格”时 up
- 大多数情况下输出 stay
- 避免频繁跳变

只输出一个单词：up/down/stay。
```

> 💡 **经验总结**：
> - **激进**：关键词触发 + 情绪词敏感 → 适合短篇叙事
> - **稳健**：多轮累积判断 → 适合长期陪伴
> - **情境**：按话题分类 → 适合多功能助手


### ⚙️ 配置项说明

#### 基础设置

| 配置 | 说明 |
| :--- | :--- |
| `启用插件` | 全局开关 |
| `人格与阶段作用域` | `session`（推荐）/ `global` |
| `总阶段数` | 2~30 阶段 |
| `阶段→人格映射` | 每行格式：`阶段编号;人格名称或ID` |
| `初始阶段` | 默认 0 |
| `启用调试日志` | 排查问题时开启 |

#### 触发设置

| 配置 | 说明 |
| :--- | :--- |
| `检测间隔（轮数）` | 每 N 轮检测一次，默认 10 |
| `启用 AI 判断` | 关闭后仅靠手动触发 |
| `辅助 LLM 模型` | 推荐选择快速模型 |
| `忽略工具调用消息` | 判断时不包含工具消息，节省 token |
| `忽略助手回复消息` | 判断时只保留用户消息（极端省 token） |

#### 手动控制

| 配置 | 说明 |
| :--- | :--- |
| `启用关键词触发` | 开启后识别 `/进阶` 等命令 |
| `进阶/降级/清空计数关键词` | 可自定义 |
| `启用工具调用` | 允许 LLM 自然语言调用工具 |
| `手动切换重置轮数` | 手动切换后重新计数 |
| `切换人设时自动清除上下文` | 适合“失忆/觉醒”类场景 |

#### 权限控制与反馈

| 配置 | 说明 |
| :--- | :--- |
| `启用权限控制` | 仅白名单用户可切换 |
| `允许的用户 ID` | QQ 号列表 |
| 各类反馈消息 | 权限不足/成功/失败/最大/最小阶段均可自定义 |


### 📋 快速上手

**1. 安装插件** → 放入 `data/plugins/`，重启 KiraAI

**2. 创建人格** → 在 KiraAI WebUI 人格管理中创建多个人格（如 `default`、`warm`、`hot`）

**3. 配置映射** → 插件设置 → 阶段人格映射：
```
0;default
1;warm
2;hot
```

**4. 测试切换** → 在会话中发送 `/进阶`，观察人格变化

### 📌 常见问题

**Q：切换到新人格后，AI 还在用旧人格语气？**

A：检查是否开启调试日志，查看 `已注入 session 人格: xxx (阶段 N)` 是否出现。若未出现，检查 `stage_persona_map` 中的名称/ID 是否与人格管理中的一致。

**Q：AI 判断一直输出 stay，从不切换？**

A：检查：
1. `enable_ai_judge` 是否开启
2. `auxiliary_model` 是否选择了有效模型
3. `trigger_interval` 是否太大（如 50 轮才触发一次）
4. 开启 `enable_debug_log`，观察 AI 判断结果

**Q：权限控制开启后，所有人包括白名单都无法切换？**

A：检查 `allowed_users` 中是否填写了正确的 QQ 号，注意不要有多余空格。

**Q：开启“自动清除上下文”后，AI 完全失忆了？**

A：是的，这是设计如此。该功能会调用 `delete_session` 清除历史，适合“失忆/觉醒”类剧情。如需保留部分记忆，请关闭此功能。
